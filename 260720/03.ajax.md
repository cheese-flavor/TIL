### 비동기 ###
```
동기: 프로그램 실행 흐름이 순차적으로 진행 = 하나의 작업이 완료된 후에 다음 작업이 실행되는 방식
       -> 이전 결과들을 기반으로 작업을 수행해야할 때 동기 

비동기: 실행흐름이 순차적이지 않으며, 작업이 완료되기를 기다리지 않고 동시에 수행하는
       -> 병렬적 수행!

block = 완료되기 전까지 막혀있는 / 제어권이 해당 
non-block = 제어권을 넘겨주는


동기 block: 늘
동기 non: 파일 설치, 작업완료되는거 확인하는
비동기 block: 문제 발생하면
비동기 non: 일 맡기고 내 일을 따로 하고있는


동시성: 한명한테 동시에 여러개 시키는거
병렬성: 여러명한테 시키는거
```

### AJAX ###
```
XMLhttpRequest 기술을 사용해서 복잡하고 동적인 프로그래밍 -> xhr
비동기적인 웹 애플리케이션 개발 기술
브라우저와 서버간의 데이터를 비동기적으로 교환
페이지 전체 새로고침 안해도 가능 -> html 페이지 일부 dom만 업데이트

단점: 코드의 실행순서가 불명확  -> 콜백 함수를 사용!
     -> 콜백 지옥


프로미스 = 콜백 지옥 문제를 해결하기 위함 (성공하면 then, 실패하면 catch)
          비동기 작업의 순차성! -> 가독성up


Axios = javascript에서 사용되는 프로미스 기반으로 동작되는 라이브러리
        외부 라이브러리여서 가져와야함 (정상적인 라이브러리를 가져왔는지 꼭!)

        기본 골격
        axios(설정객체)
            .then(성공했을때 실행할 콜백)
            .catch(실패했을때 실행할 콜백)

        ex.
        axios({
            method: 'get'     -> 요청방식
            url: '~'          -> 요청을 보낼 주소
            params: {         -> 쿼리 스트링 / key:value값
                name: name
            }
        })
        .then((response) => {
            consol.log(response.data.age)  -> 객체를 자동으로 넘겨준다? #response가 큰 봉투(여러 정보가담긴)
        })
        .catch((error) => {   -> 실패했을때 나타남
            console.error(error)
        })

```
